内核与发行版

内核

  • 系统核心，一般包括5大部分
    • 进程管理
    • 存储管理
    • 文件管理
    • 设备管理
    • 网络管理

发行版

  • 以内核为中心，集成各种系统管理软件应用工具组成的一套操作系统，称为Linux发行版

    • CentOS（本文主要以CentOS为例进行说明)
      • RedHat的社区版
      • 稳定
    • Ubuntu
      • 安装简单
      • 界面华丽
      • 驱动支持
      • 社区活跃
      • 更新频繁
      • 基于Debian
    • Debian
      • 软件包通常是稳定的，而不是最新的
      • 社区维护
      • apt软件包管理
      • 默认不包含任何专业驱动
    • Oracle Linux
      • 专门为Oracle数据库进行优化，性能提高迅猛
      • 基于RedHat
    • openSUSE
      • 没用过

  • 各发行版间的包管理器

    • Debian
      • apt
    • CentOS
      • yum
    • openSUSE
      • YaST

  • 环境变量文件
    • /etc/profile
      • 用户登录或su -命令切换时执行，Login shell执行，Non-login shell不执行
      • 全局变量
      • 循环执行/etc/profile.d中的脚本，profile.d目录下的脚本文件包括各应用的初始配置
    • /etc/bashrc
      • 执行完/etc/profile后运行bash shell就会执行
      • 每执行一次bash shell执行一次
    • ~/.bash_profile
      • /etc/profile的用户级别设置
      • Login shell执行，Non-login shell不执行
      • 默认情况会执行用户的~/.bashrc
    • ~/.bashrc
      • 用户级别设置，打开新shell时执行
    • ~/.bash_logout
      • 退出shell时执行
      • 删除临时文件等

补充一波wget

#爬个虫
wget -r -p -nc -np -k -c --progress=bar http://www.dcdapp.com

#帮助文档
wget --help
  • 部分参数说明
    • -r –recursive指定递归下载
    • -p –page-requisites下载所有用于显示 HTML 页面的图片之类的元素。
    • -nc –no-clobber skip downloads that would download to existing files (overwriting them).
    • -np–no-parent 不追溯至父目录。
    • -k–convert-links 让下载得到的 HTML 或 CSS 中的链接指向本地文件。
    • -c –continue 断点续传下载文件。–progress=TYPE 选择进度条类型。
    • -b –background 启动后转入后台。
    • -O –output-document=FILE 将文档写入 FILE。
    • -P –directory-prefix=PREFIX 以 PREFIX/… 保存文件
    • -t –tries=NUMBER 设置重试次数为 NUMBER (0 代表无限制)。

介绍了wget，怎能不介绍curl

  • 常用参数

    • -o指定文件名
curl -o #1_#2.tar.gz http://www.xxxx.com/{a1,a2}/xo[1-5].tar.gz
  • -O–remote-name Write output to a file named as the remote file
curl -O http://www.xxxx.com/{a1,a2}/xo[1-5].tar.gz

#补充一个grep的转义，字符-需要使用\进行转义，并且查询内容要用单引号框起来才能查询出结果，如下所示：
curl --help|grep '\-O'
-O, --remote-name   Write output to a file named as the remote file
  • -r 分块下载，–range RANGE Retrieve only the bytes within a range
curl -r 0-10000 -o file_part1.tar.gz http://www.xxx.com/xxx.tar.gz
curl -r 10000-20000 -o file_part2.tar.gz http://www.xxx.com/xxx.tar.gz
curl -r 20000- -o file_part3.tar.gz http://www.xxx.com/xxx.tar.gz
cat file_part* > file.tar.gz
  • -T–upload-file FILE Transfer FILE to destination
  • -F –form <name=content> 模拟http表单提交数据
  • -c–cookie-jar-file 操作结束后把cookie写入到这个文件中
  • -D 保存头信息
curl -c ./ck_c.txt -D ./ck_d.txt -F name=xxx -F pass=yyy http://www.xoxo.com/login
curl -T pic=@localfile http://www.xoxo.com/upload
  • -A随意指定当前浏览器信息
  • -b使用cookie文件
  • -C断点续传
  • -d使用POST
curl -d "name=xxx&pass=yyy" http://www.xoxo.com/login
  • -v verbose，打印更多信息
  • -m –max-time 指定处理的最大时长
  • -H指定请求头
  • -H–header
    指定请求头参数
  • -s–slient 减少输出的信息
  • --connect-timeout指定尝试连接的最大时长
  • -x/--proxy指定代理服务器地址和端口，端口默认为1080
  • --retry 指定重试次数
  • -e/--referer 指定引用地址
  • -I/--head 仅返回头部信息，使用HEAD请求
  • -u/--user设置服务器的用户和密码

打包完成后，不知道文件放哪里了，需要使用find命令

  • 常用参数
参数名 描述 参数名 描述
-newer 文件最近一次修改比 file 修改时间要晚 -perm 按执行权限来查找
-anewer 文件最近一次访问比 file 修改时间要晚 -type 按照文件类型来查找
-cnewer 文件状态最近一次修改比 file 修改时间要晚 -size 按照文件的大小来进行查找
-amin 文件最近一次访问是在 n 分钟之前 -user 按文件所属用户来查找
-atime 文件最近一次访问是在 n*24 小时之前 -group 按文件所属用户组来查找
-cmin 文件状态最近一次修改是在 n 分钟之前 -nouser 无有效所属用户的文件
-ctime 文件状态最近一次修改是在 n*24 小时之前 -nogroup 无有效所属用户组的文件
-mmin 文件内容最近一次修改是在 n 分钟之前 -empty 空的普通文件或者空目录
-mtime 文件内容最近一次修改是在 n*24 小时之前

重定向

  • 概念

    • 标准输入 stdin 文件描述符0

    • 标准输出 stdout 文件描述符1

    • 错误输出 stderr 文件描述符 2

    • 0 1 2是如何定义的

[sulin@localhost ~]$ ll /dev/std*
lrwxrwxrwx. 1 root root 15 10月 17 00:22 /dev/stderr -> /proc/self/fd/2
lrwxrwxrwx. 1 root root 15 10月 17 00:22 /dev/stdin -> /proc/self/fd/0
lrwxrwxrwx. 1 root root 15 10月 17 00:22 /dev/stdout -> /proc/self/fd/1

  • 示例

    • 输出重定向
#标准输出重定向到文件，覆盖文件
ls -l > log.txt
#追加到文件
ls -l >> log.txt
#将错误输出重定向到文件，覆盖文件
commond 2 > err.txt
#追加
commond 2 >> err.txt
#输出追加到log，错误重定向到输出
commond >> log.txt 2>&1
#同上
commond & >> log.txt
  • 输入重定向
#将文件作为命令的标准输入
commond < file
#从标准输入读，直到分界符停止
commond << EOF
#文件1作为标准输入，并重定向输出到文件2
commond < file1 > file2
  • /dev/null

网络命令

  • 端口

    • netstat -nltp

  • 防火墙

    • 使用内置防火墙
#临时操作
systemctl status firewalld
systemctl start firewalld
systemctl stop firewalld
#永久操作，开机启动
systemctl enable firewalld
systemctl disable firewalld
#打开关闭端口
firewall-cmd --zone=public --add-port=8080/tcp --permanent
firewall-cmd --zone=public --remove-port=8080/tcp --permanent
#配置立即生效
firewall-cmd --reload
#查看打开的端口
firewall-cmd --zone=public --list-ports
#防火墙状态
firewall-cmd --state
#centos7默认没有 netstat 命令，需要安装 net-tools 工具，yum install -y net-tools
netstat -nltp

#区域相关
firewall-cmd --list-all-zones #查看所有区域信息
firewall-cmd --get-active-zones #查看活动区域信息
firewall-cmd --set-default-zone=public #设置public为默认区域
firewall-cmd --get-default-zone #查看默认区域信息
firewall-cmd --zone=public --add-interface=eth0 #将接口eth0加入区域public
#接口相关
firewall-cmd --zone=public --remove-interface=eth0 #从区域public中删除接口eth0
firewall-cmd --zone=default --change-interface=eth0 #修改接口eth0所属区域为default
firewall-cmd --get-zone-of-interface=eth0 #查看接口eth0所属区域

#路由转发
# 将80端口的流量转发至8000
firewall-cmd --add-forward-port=port=80:proto=tcp:toport=8000   
# 将80端口的流量转发至10.10.0.1
firewall-cmd --add-forward-port=proto=80:proto=tcp:toaddr=10.10.0.1 
# 将80端口的流量转发至10.10.0.1:8000
firewall-cmd --add-forward-port=proto=80:proto=tcp:toaddr=10.10.0.1:toport=8000

# 检查是否允许伪装IP
firewall-cmd --query-masquerade  
# 允许防火墙伪装IP
firewall-cmd --add-masquerade    
# 禁止防火墙伪装IP
firewall-cmd --remove-masquerade

#rich rule
firewall-cmd --permanent --zone=public --add-rich-rule='rule family="ipv4" source address="192.168.0.4/24" service name="http" accept'
#删除--remove-rich-rule
  • 使用iptables
#检查状态
service iptables status
#安装
yum install -y iptables
#查看端口
/sbin/iptables -L -n
#生成/etc/sysconfig/iptables
iptables -P INPUT ACCESS
#开放某个端口
/sbin/iptables -I INPUT -p tcp --dport 8080 -j ACCEPT
#保存
iptable-save > /etc/sysconfig/iptables
#检查
netstat -an |grep 8080
  • firewall和iptable冲突，需要禁用一个
#停止iptables服务
systemctl stop iptables
#禁用iptables服务
systemctl mask iptables

  • 区别

    firewall是centos7开始的新命令，底层还是iptable
